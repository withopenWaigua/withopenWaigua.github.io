Java封装性与访问控制&&多态详解

Posted on 2021-12-06 | In Java , 基础内容

封装性浅析

  • 封装、继承、多态

封装性就是通过设置适当的访问级别来限制对我们的访问

  • 该隐藏的隐藏，该暴露的暴露

访问控制权限等级和访问控制符

  • 四个等级，从最大到最小权限
  • public 公共访问权限 【公共访问权限】
  • proteced 保护的【子类访问权限】
  • 啥也不屑：缺省的 【包访问权限】
  • private:私有的 【类访问权限】

定义类： 【修饰符】 class 类名{}

定义变量： 【修饰符】 类型 成员变量名；

定义方法 【修饰符】返回类型 方法名（）{}

public

  • 成员变量、方法、类用public修饰，那么就可以被所有类访问

    • 不管访问的类或者被访问的类是否都处于一个包中，是否有父子关系，都无所谓

    • 一般能够被外界访问的东西我们定义为public

protected

  • 不能挎包了 全局访问就是挎包

啥也不写 缺省的

  • 缺省的不能跨包了
  • 只能同一个包
  • 挎包后子类也不能调用父类的变量了

private

  • 只能类内访问

类的权限等级

  • 定义一个类只能用到public和缺省的
  • 如果该类被用public修饰，那么这个类就能被其他类使用，如果上面都没有，那么这个类只能被同一个包中的其他类使用
  • 源文件和类名的关系
    • 如果一个源文件多个类，前面没用public修饰，那么java源文件名任意取
    • 如果有一个public类，那么这个源文件必须用public类名命名
    • 一个java源码中最多定义一个public类

访问控制符：解决类的开发者和使用这之间的关系问题

类设计的原则

  1. 不需要对外使用的成员变量和方法，全部用private修饰
  2. 对于父类中的方法，需要在子类进行覆盖重写，父类习惯使用protected
  3. 希望暴露给其他类使用的就要public
  4. 我们创建一个类对象一般都要调用类的构造器，所以构造器一般要写成public

多态详解

  • 父类指针【引用】指向子类对象，是我们探讨多态性的前提

编译类型，运行类型

  • 类对象是一种引用类型变量，像一个指针，指向堆中的一块表示对象的内存
  • 编译型类型：声明对象时候的类型，在编译时候就能固定下来
  • 运行时类型：程序运行起来，那么对象实际指向堆内存中的对象类型，就是对象的运行时类型
  • myhuman2 编译的时候是Human父类，运行时候是Men类型

多态性的清晰解释

  1. 对象的编译类型是父类类型，对象的运行类型是子类类型
  2. 父类中有一个方法，但子类覆盖了父类中的该同名方法
    1. 父类子类方法名字相同
    2. 参数相同
    3. 返回类型相同或者兼容
      • 兼容：比如说父类中这个方法返回的是个类类型，那么返回这个个类类型的子类型；
  3. 方法myfunc不能是静态的，不能是static静态的
  4. 通过对象名.方法名来调用这个方法

多态性解释

  • 多态性体现在具有继承关系的父类和子类，子类覆盖了父类的方法myfunc（）
  • 通过父类对象名，只有到程序运行时，找到该父类对象所指向的真正堆内存中类对象实例，然后系统内部根据堆对象实例类型，找到并调用父类或子类的myfunc方法
  • 调用同一个方法，会执行不同代码段 这种特征就是运行期间的多态性

对象的强制转换

  • 同子类不能转

instanceof

  • 判断是否指向它的父或本类，因为同子类不能转
  • 格式：对象名 instanceof 类名
  • 返回bool true/false
  • 大类型转小类型总是可以成功

调用一个多态方法，却决于类类型引用所指向的真实类对象实例，指向那个类对象实例，就执行那个类多态方法
