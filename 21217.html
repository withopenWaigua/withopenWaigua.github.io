Java初始化块详解以及抽象类与抽象方法

Posted on 2021-12-06 | In Java , 基础内容

初始化块

  • 用{}包含初始化内容的块
  • 初始化块先执行，再执行构造器
  • 类中成员变量的定义并初始化代码和初始化代码块里面的执行是平级的，也就是再类定义中，谁排前面就先执行谁。谁排后面就后执行谁。后面可以换前面的

new类对象：

  1. 父类初始化块
  2. 父类构造器
  3. 子类初始化块
  4. 子类构造器

静态初始化块

多了static

  • 静态初始化块先执行
  • 先定义的先执行
  • 普通初始化块

new类对象：

  1. 父类的静态初始化块
  2. 子类的静态初始化块
  3. 父类初始化块
  4. 父类构造器
  5. 子类初始化块
  6. 子类构造器
  • 类初始化块只能初始化一次
  • 普通初始化块，对象初始化多少次，普通就执行几次
  • 类初始化块只能引用静态成员变量和静态方法

构造器

  • 当生成子类时候父类构造器中执行有子类和父类同名一方法，那么父类构造器执行的是子类里面的方法
  • 不要在父类构造器引用被子类覆盖的方法

抽象类和抽象方法

  • 抽象类—类
  • 抽出概括出他们共同的方面、本质属性与关系
  • 把一些共通的东西抽取出来
  • 抽象类一般用来作为父类，把一些公共的东西抽象出来
  • 只需要在类定义前面加入abstract

抽象方法

  • 把函数返回前面加入abstract
  • 不能有方法体，并且方法后面加’；’
  • 可以理解为抽象方法是一种不完整的方法，只有声明没有方法体

抽象类和抽象方法的特性

  • 都用abstract修饰，同时抽象方法不能有方法体

  • 抽象类不能用new来实例化；

  • 抽象类里面不一定要有抽象方法，只要增加了abstract就变成抽象类

  • 抽象类也可以有普通成员变量，普通方法，初始化块，也能有静态成员变量，静态方法，静态初始化块以及自己的构造器

  • 抽象类无法实例化，所以构造器主要是提供给子类使用

  • 如果一个方法是抽象方法，那么这个类必须是抽象类

  • 普通方法不能包含抽象方法

抽象类的使用

  • 抽象类被用于父类继承
  • 子类继承父类之后必须把父类中的抽象方法实现掉
  • 在子类中实现了父类的抽象方法，子类也不再是抽象类

总结：

  • 抽象类，被用于继承
  • 必须在子类中重写
  • 只能在普通方法之前加入abstract，末尾加;
  • 静态方法，构造器不能加入abstract
  • 子类要覆盖父类方法，所以父类不能设置private的抽象方法

抽象类引入目的探讨

  • 抽象类不能创建实例，但可以当作父类取创建子类，然后把一些公共接口写成抽象方法，弄到抽象类中取，然后所继承的子类重写这些抽象方法。
  • 抽象类是给子类指定一个书写规范(书写模板)，要求子类必须要重写父类的抽象方法
