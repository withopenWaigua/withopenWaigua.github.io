Java内部类解析

Posted on 2021-12-06 | In Java , 基础内容

内部类

  • 内部类（嵌套类）
  • 外部类（相当于其里面的类）

内部类的基本使用

类：

  • 方法
  • 构造器
  • 成员变量
  • 初始化块
  • 内部类
  1. 外部类能访问内部类的私有成员变量
  2. 内部类也可以访问外部类的私有方法、私有成员变量

变量的访问顺序

  1. 局部变量优先（成员变量名）
  2. 本类中的成员变量（this+成员变量名）
  3. 外部类的成员变量(外部类.this.成员变量名)
  • 内部类的方法能直接访问外部类的成员变量（直接成员变量名）
  • 外部类需要用 对象名.变量名 来访问内部类成员变量

为什么内部类能直接访问外部成员变量而外部类不行

  • 成员变量跟着对象走,有对象才有成员变量.
  • 有内部类对象存在，一定有外部类对象存在
  • 能调用内部类的存在，一定有一个内部类存在
  • 如果有一个内部类存在，那么一定有一个外部类存在
  • 所以内部类方法中，能直接访问外部类对象成员变量
  • 相反，有外部类存在，不一定有内部类存在，所以得先创建一个内部类对象去调用它

静态规则

  • 不允许在外部类的静态方法中使用内部类
  • 一般的内部类不要沾染上和“静态（static）”有关的内容，不然会报错

内部类种类

  1. 普通内部类
    • 类内定义另外一个类，外部类访问内部类和访问一个普通类区别不大
  2. 局部内部类
    • 作用域局限在定义它的方法中，无法在外部类其他方法中使用
  3. 静态内部类
    • 用static修饰的内部类
    • 一旦定义一个静态内部类，那么该类就属于外部类的组成部分了(跟外部类对象没有什么关系)
    • 什么成员都可以往 静态内部类里面放（但静态类中的静态方法不能访问非静态成员变量）
    • 静态内部类中的普通方法也不要去访问外部类的普通方法，只能访问外部类的静态成员
    • 把静态类当作外部类的静态成员就行（静态方法，静态成员变量，静态初始化块）
  4. 接口中的内部类
    • 接口中是可以在内部类中定义一个静态成员变量，为什么？
      • 因为在接口中，自动会为内部类使用static修饰，也就是说接口中出现的内部类都是静态内部类
  5. 匿名类
    • 匿名内部类必须有个爹，这个爹可以是个类，也可以是个接口
    • new 接口名(){……} 或者new 类名(参数){……}
    • 匿名类属于一种一次使用的类，创建这个类的时候就自动创建了一个该类的实例，该类不能复用
    • 匿名类没用名字，所以没有自己的构造器，但可以有自己的初始化块
    • 定义匿名类不需要class关键字；这种new的写法类似有

内部类的使用

  • 和使用一个普通类没什么太大区别，一般是new一个内部类对象，然后调用内部类的方法

  • 在外部使用普通外部类

    • 格式：外部类名.内部类名 变量名；

    • Human myhuman = new Human();
myfood = new myhuman.new HumanFood();
myfood.XXX

内部类的访问权限

  • 外部类修饰：public或什么都不写
  • 内部类：public 啥也不写，protected，private
    • public：公共的，任意访问
    • protected：保护的，不可以挎包使用，但可以通过继承来实现挎包使用
    • 啥也不写：跟protected几乎一样，不能挎包，可以被与外部类同一个包中的其他类以及外部类的子类访问（前提是在同一个包下，不能挎包的子类）
    • private：只能外部类的内部使用.
    • 外部类名 $ 内部类名.class

Java的final用法详解&接口详解

Posted on 2021-12-06 | In Java , 基础内容

final

  • 修饰符/关键字，可以修饰
    • 成员变量（普通成员变量、静态成员变量） 局部变量(形参)
    • 方法
    • 类
  • 最终的，不能改变

final修饰变量

  • 特性是一旦有了初值之后，值就不能改变
  • 成员变量
    • 普通成员变量定义的时候给初值，或者初始化块给，构造器给
    • 如果创建了对象，可以在外部给，不给初始值也有缺省值
    • 整型：0，Boolean=false，Human =null;
  • final必须要给初始值-成员变量
    • 定义的时候给初始值
    • 初始化块给初值
    • 构造器给初值
    • 但上面就只能在一个地方定值
  • 静态成员变量
    • 在定义的时候给初始值
    • 在静态初始化块给初值
    • 但上面就只能在一个地方定值
  • 局部变量
    • final修饰的普通局部变量
      • 可以在定义的时候给初值，也可以在定义的时候不给初值，后续赋值语句给值，但一旦给了值就不可以赋给其他值了
    • final修饰的形参
      • 形参的值是通过方法传递的实参值来给的
    • final修饰的引用类型
      • final修饰的引用类型局部变量，一旦指向了某块堆内存，就不允许在指向其他堆内存了，指向位置不能改变，但指向的值可以改变

final宏替换能力

  • 为什么两个字符串String内容相同，比较结果不相同
  • 有的字符串的内容/值在编译的时候就能确定
    • 而result编译的时候确定不了其值，而需要在运行的时候才能确定result3的值
  • Java针对字符串，比如编译期间就能确定字符串内容的，这些内容会被缓存到一个专门的地方去
  • sr = result = result2
  • 如果用final那么：sr = result = result2 = result3 = result4
  • 宏替换在编译就能缓存值

final修饰方法

  • 一旦一个方法被final修饰，那么这个方法就不可以被子类覆盖了
  • private 不要和final 联合修饰，因为private不会被继承

final修饰类

  • 就不能继承这个类了

接口的基本概念

  • 接口：比抽象类还抽象
  • 这里把接口理解为抽象类 —接口理解为类
  • 格式：
    • 【修饰符】 interface 接口名{
    • }
  • 修饰符一般是public，不加就是缺省访问权限

接口的使用

  • 接口也不能实例化

  • 接口内部一般定义一些规范的东西

  • 规定继承者必须要实现的东西

  • 一般是用具体的类去实现这个接口（当父类）。实现这个接口的类，叫做实现类

  • ```java
    public class impint implements kxinterface{ //实现类

    }
    //继承两个类
    public class extends Human impint implements kxinterface{ //实现类

    }
    //继承多个接口
    public class impint implements kxinterface,kxinterface2{ //实现类

    }

  • 接口不能实例化，和抽象类一样

  • 接口可被用于声明[定义] 引用类型变量;多态

接口核心思想和可放入内容

  • 接口一般定义和规范性的东西，但不提供具体的实现。规范和实现分类的设计原则
    • 规范：接口中
    • 实现：在具体的实现类中

接口中不允许出现的内容

  • 构造器
  • 初始化块
  • 接口可以继承多个接口但不可以继承类
  • 接口中不能出现普通成员变量（只有静态成员变量），没有普通方法(有方法体的方法)，只有抽象方法（没用方法体）

共性问题

  • 接口里面大部分都是public权限。所以接口定义的内容可以不写public，但是要写就必须要写成public，写成其他修饰符会报语法错误

接口允许的内容

  • 成员变量（自动变成了静态成员变量）
    • 自动默认加上static 和final
    • static：跟着类走
    • final：必须给这个成员变量指定初始值，而且一旦有初始值就不能修改
  • 普通成员方法（自动变成抽象方法）
    • 不写任何前面的修饰词
    • 自动加上了 abstract，不可以有方法体
    • 必须要实现接口里面的方法
    • 不是静态方法，不能用接口调用，只能用对象（多态（父=子）或（子=子））
  • 静态方法（变成类方法）
    • 跟着类走，用static修饰。
    • 必须要有方法体
    • 直接用接口调用（不能用对象调用）
  • 默认方法（实例方法）
    • 用default修饰，public可以省略，必须要有方法体，否则报错
    • 跟着对象走的，所以不可以和static连用
    • 能写方法体
  • 私有方法
    • 用private修饰
    • 必须要有方法体
    • 跟着对象走（普通私有方法）
    • 跟着类走（加上static）
    • 无法被main调用，作用是仅供接口内部调用
    • 不能被普通方法调用

接口的继承

  • 一个接口多个父类，但接口只能继承接口，不能继承类

接口与抽象类的概括

  • 抽象类是从多个类中抽取出来的模板，接口。接口好像是100%纯抽象类。
  • 接口是一种设计规范，具体的实现是通过继承这个接口的实现类来实现的，私有接口体现的是一种规范和实现分离的逻辑
  • 接口，可以看作一个类，只不过这个类比较特殊
  • 接口的主要途径是作为实现类的父类。并在具体实现类中实现需要的东西，比如接口中的普通方法（抽象方法）

接口和抽象类的一同

  • 两者都不能通过实例化，不如new一个接口，new一个抽象类都不行
  • 抽象类一般通过子类的继承extends来使用
  • 接口通过一个类的实现implements来实现。所以抽象类和接口都是等着别人来几次
  • 有些接口里可以有，抽象类不可以有，反之亦然
  • 接口中出现的很多东西，都是自带的public和static修饰

Java初始化块详解以及抽象类与抽象方法

Posted on 2021-12-06 | In Java , 基础内容

初始化块

  • 用{}包含初始化内容的块
  • 初始化块先执行，再执行构造器
  • 类中成员变量的定义并初始化代码和初始化代码块里面的执行是平级的，也就是再类定义中，谁排前面就先执行谁。谁排后面就后执行谁。后面可以换前面的

new类对象：

  1. 父类初始化块
  2. 父类构造器
  3. 子类初始化块
  4. 子类构造器

静态初始化块

多了static

  • 静态初始化块先执行
  • 先定义的先执行
  • 普通初始化块

new类对象：

  1. 父类的静态初始化块
  2. 子类的静态初始化块
  3. 父类初始化块
  4. 父类构造器
  5. 子类初始化块
  6. 子类构造器
  • 类初始化块只能初始化一次
  • 普通初始化块，对象初始化多少次，普通就执行几次
  • 类初始化块只能引用静态成员变量和静态方法

构造器

  • 当生成子类时候父类构造器中执行有子类和父类同名一方法，那么父类构造器执行的是子类里面的方法
  • 不要在父类构造器引用被子类覆盖的方法

抽象类和抽象方法

  • 抽象类—类

  • 抽搐概括出他们共同的方面、本质属性与关系

  • 把一些共通的东西抽取出来

  • 抽象类一般用来作为父类，把一些公共的东西抽象出来

  • 只需要在类定义前面加入abstract

  • ```java
    public abstract class Human {

    }

  • 不能有方法体，并且方法后面加’；’

  • 可以理解为抽象方法是一种不完整的方法，只有声明没有方法体

抽象类和抽象方法的特性

  • 都用abstract修饰，同时抽象方法不能有方法体

  • 抽象类不能用new来实例化；

  • 抽象类里面不一定要有抽象方法，只要增加了abstract就变成抽象类

  • 抽象类也可以有普通成员变量，普通方法，初始化块，也能有静态成员变量，静态方法，静态初始化块以及自己的构造器

  • 抽象类无法实例化，所以构造器主要是提供给子类使用

  • 如果一个方法是抽象方法，那么这个类必须是抽象类

  • 普通方法不能包含抽象方法

抽象类的使用

  • 抽象类被用于父类继承
  • 子类继承父类之后必须把父类中的抽象方法实现掉
  • 在子类中实现了父类的抽象方法，子类也不再是抽象类

总结：

  • 抽象类，被用于继承
  • 必须在子类中重写
  • 只能在普通方法之前加入abstract，末尾加;
  • 静态方法，构造器不能加入abstract
  • 子类要覆盖父类方法，所以父类不能设置private的抽象方法

抽象类引入目的探讨

  • 抽象类不能创建实例，但可以当作父类取创建子类，然后把一些公共接口写成抽象方法，弄到抽象类中取，然后所继承的子类重写这些抽象方法。
  • 抽象类是给子类指定一个书写规范(书写模板)，要求子类必须要重写父类的抽象方法

Java封装性与访问控制&&多态详解

Posted on 2021-12-06 | In Java , 基础内容

封装性浅析

  • 封装、继承、多态

封装性就是通过设置适当的访问级别来限制对我们的访问

  • 该隐藏的隐藏，该暴露的暴露

访问控制权限等级和访问控制符

  • 四个等级，从最大到最小权限
  • public 公共访问权限 【公共访问权限】
  • proteced 保护的【子类访问权限】
  • 啥也不屑：缺省的 【包访问权限】
  • private:私有的 【类访问权限】

定义类： 【修饰符】 class 类名{}

定义变量： 【修饰符】 类型 成员变量名；

定义方法 【修饰符】返回类型 方法名（）{}

public

  • 成员变量、方法、类用public修饰，那么就可以被所有类访问

    • 不管访问的类或者被访问的类是否都处于一个包中，是否有父子关系，都无所谓

    • ```java
      public int Hour = 0;
      // main
      mpgt.Time mytime = new mpgt.Time();
      mytime.Hour=12;//不报错

啥也不写 缺省的

  • 缺省的不能跨包了
  • 只能同一个包
  • 挎包后子类也不能调用父类的变量了

private

  • 只能类内访问

类的权限等级

  • 定义一个类只能用到public和缺省的
  • 如果该类被用public修饰，那么这个类就能被其他类使用，如果上面都没有，那么这个类只能被同一个包中的其他类使用
  • 源文件和类名的关系
    • 如果一个源文件多个类，前面没用public修饰，那么java源文件名任意取
    • 如果有一个public类，那么这个源文件必须用public类名命名
    • 一个java源码中最多定义一个public类

访问控制符：解决类的开发者和使用这之间的关系问题

类设计的原则

  1. 不需要对外使用的成员变量和方法，全部用private修饰
  2. 对于父类中的方法，需要在子类进行覆盖重写，父类习惯使用protected
  3. 希望暴露给其他类使用的就要public
  4. 我们创建一个类对象一般都要调用类的构造器，所以构造器一般要写成public

多态详解

  • 父类指针【引用】指向子类对象，是我们探讨多态性的前提

编译类型，运行类型

  • 类对象是一种引用类型变量，像一个指针，指向堆中的一块表示对象的内存
  • 编译型类型：声明对象时候的类型，在编译时候就能固定下来
  • 运行时类型：程序运行起来，那么对象实际指向堆内存中的对象类型，就是对象的运行时类型
  • myhuman2 编译的时候是Human父类，运行时候是Men类型

多态性的清晰解释

  1. 对象的编译类型是父类类型，对象的运行类型是子类类型
  2. 父类中有一个方法，但子类覆盖了父类中的该同名方法
    1. 父类子类方法名字相同
    2. 参数相同
    3. 返回类型相同或者兼容
      • 兼容：比如说父类中这个方法返回的是个类类型，那么返回这个个类类型的子类型；
  3. 方法myfunc不能是静态的，不能是static静态的
  4. 通过对象名.方法名来调用这个方法

多态性解释

  • 多态性体现在具有继承关系的父类和子类，子类覆盖了父类的方法myfunc（）
  • 通过父类对象名，只有到程序运行时，找到该父类对象所指向的真正堆内存中类对象实例，然后系统内部根据堆对象实例类型，找到并调用父类或子类的myfunc方法
  • 调用同一个方法，会执行不同代码段 这种特征就是运行期间的多态性

对象的强制转换

  • 同子类不能转

instanceof

  • 判断是否指向它的父或本类，因为同子类不能转
  • 格式：对象名 instanceof 类名
  • 返回bool true/false
  • 大类型转小类型总是可以成功

调用一个多态方法，却决于类类型引用所指向的真实类对象实例，指向那个类对象实例，就执行那个类多态方法

Java的package和import详解

Posted on 2021-12-06 | In Java , 基础内容

package

  • 解决类名重复的问题
  • 为了区别相同的类名，引入package，提供一个所谓的【名字空间、命名空间】，我们可以把一堆、一组类放到这个包名之下
  • 我们可以通过package把一组类组织带一起，让这些类归属到一个package下，就是归属再这个包名之下
  • 格式： package 包名;
  • 写到java源文件第一行，非注释行位置
  • 这样该源文件中的所有类都包含在类名上面
  • 如果使用了package抱住了一个或多个类，目录结构也要和包名层次相同
  • 一个源文件里面定义的类，只能定义到一个包里
  • 不指定任何pacakage，就被放到一个默认的package

命名：

  • 推荐以自己域名反过来命名 com.baidu.www
  • 但变成的是 com/baidu/www

注意：

  • 同一个包里面的类可以自由通过类名来直接访问

  • 不同包里的类不能通过类名直接访问

    • 通过包名.类名来引用类

    • ```java
      mpgt.Time mytime = new mpgt.Time();

  • 导入包类的全部类

  • import mpgt.*;

  • import 依旧冲突就必须包名.类名

import 静态导入

  • 导入某个静态成员变量、方法，甚至导入每个类的全部静态成员变量和方法

导入某个类中的单个静态成员变量、方法

格式：

  • import static 包名.类名.静态成员变量名|静态方法
  • 导入全部静态成员变量和方法
  • import static 包名.类名.*

import和import静态导入的区别

常规的import主要作用是让我们免去写包名之苦，只写类名

  • import static更里河，使用之后连类名都不需要写了，直接写静态成员变量名、静态方法名

System.out.println();

  • java.lang 是个包名，具有多个类，默认调入了全部的类，无需手动导入

如何进行类继承

父亲类、孩子类；

父类【基类、超类】派生出孩子类【派生类、子类】

  • 既有父亲类又有孩子类这种层次关系我们就叫做继承，孩子类能从父亲类哪里继承到许多东西
  • 我们可以把一些通用的东西放在父类，然后每个孩子可以派生出自己独特的东西，还可以从父亲哪里继承出一些通用的东西，防止子类中出现很多重复的代码

继承

  1. 先定义一个父类
  2. 父类定义一些公共的成员变量、方法
  3. 通过继承这个父类来构建新的子类
  • 通常来说，子类会比他的父类有更多的成员变量以及方法，子类一般比父类更庞大【内容更多，占用更多内存】

子类继承父类的语法格式：

修饰符 class 子类名 extends 父类名{

//成员变量

//成员方法

}

  • extends 扩展
  • 只能继承一个父类
  • 继承后，子对象包含了父类的对象

覆盖父类的方法

  • 子类重写一个参数相同的方法（必须返回值相同）
  • 如果参数不同【数量不同，类型不同】， 那么就不会覆盖
  • 静态方法也一样用静态方法去覆盖

调用父类同名函数的方法

  • super - 》理解成指向当前对象的父亲子对象的箭头或指针，super一般也只出现在类的方法中
  • this -》指向堆中的自己
  • 静态方法中不能出现this，super在静态方法中也不能够使用
  • 有对象才有this、super

覆盖父类的成员变量

  • 在java的类继承中，子类能覆盖父类的同名变量

如果覆盖后还要使用父类的成员变量

  • 强制转换
  • 可以强转调用父类中的成员变量 /静态成员变量/被覆盖的静态方法/未被覆盖后的普通方法
  • 但不能强转成覆盖后的父类的普通方法

构造器

  • 定义并初始化一个对象时候

  • 父类构造器先执行

  • 再执行了子类构造器

  • public class Human {
    //构造器
    Human(int i){
        System.out.println("执行了Human::human()构造器");
    }
public class Men extends Human{
    Men(){
        super(12); //因为父类构造器有了参数，无法使用默认构造器，因此要子类传参数进去调用父类构造器，还必须写道第一行
        System.out.println("执行了Men()构造器");
    }

Men mymen = new Men();

结论：

  1. 不管是否第一行使用了super（）来调用父类构造器，在初始化一个Men对象的时候，父类和子类构造器都只执行一次
  2. 不写的时候默认调用父类的默认构造器，要调用其他构造器就必须写super（XXXX）
  3. 既然可以通过super（）调用父类构造器，可以推出this可以调用本类的其他构造器
  4. super和this不能同时出现，因为都必须放在子类构造器第一行
  5. 如果子类构造器没用super和this，先调用父类构造器，再调用子类构造器

Java构造器、this以及static关键字细说

Posted on 2021-12-06 | In Java , 基础内容

构造器细说

  • 构造器其实就是一个方法【成员函数】，构造器有一些特殊之处
  • 名字特殊：必须与类名相同
  • 构造器不要写返回类型不能写返回值return，以及加入void，加入void就是变成了成员函数而不是构造器了
  • 构造器的能力就是当你输出一个对象的时候，构造器会被自动调用

默认构造器：不带参数的构造器

一个类能有多个构造器，只要各个构造器要么是参数数量不同或者参数类型不同就行

报错

  • 如果一个类中没有我们自己书写的构造器，那么系统会为该类提供一个默认的构造器
    • 如果有了任意一个构造器，就不能调用默认的构造器，不使用自自定义的构造器，使用默认构造器就会报错
  • 如果程序员为一个类使用了构造器，那么系统不再使用默认构造器
  • 不要手工调用构造器

this关键字

this一般只能出现在类的方法中，不能出现在main中

  • this可以理解为指针、箭头、引用
  • this的真实身份是一个隐藏起来的函数参数

this含义

  • 表示调用方法的那个对象的引用
  • 指向该方法的对象

static 关键字详解

  • 静态的意思
  • 一旦使用static这个修饰，就变成了静态成员变量就属于该类的成员变量，而不是该对象的成员变量
  • 静态成员变量可以使用任意的类名和对象名进行引用；

静态成员函数

  • 用stastic修饰的成员函数
  • 不能调用和对象有关的成员变量，它只属于类而不是对象，打印都不行
  • 但可以操作静态成员变量
  • 静态成员函数不能使用this，没有this，this只有对象里面有
  • 普通方法可以调用静态方法
  • 静态方法不能调用普通方法

总结：

  • 有static修饰的属于类本身，没有static修饰的，属于类对象
  • static修饰的成员变量和方法，都可以使用类调用，也可以使用对象调用，没有static的变量和方法，只能使用对象实例来调用
  • 静态成员不能访问非静态成员，因为静态成员不支持this关键字，而非静态成员需要this才能访问
  • 尽量使用类名而不是对象名来访问和使用
