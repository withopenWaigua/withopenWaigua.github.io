Java的final用法详解&接口详解

Posted on 2021-12-06 | In Java , 基础内容

final

  • 修饰符/关键字，可以修饰
    • 成员变量（普通成员变量、静态成员变量） 局部变量(形参)
    • 方法
    • 类
  • 最终的，不能改变

final修饰变量

  • 特性是一旦有了初值之后，值就不能改变
  • 成员变量
    • 普通成员变量定义的时候给初值，或者初始化块给，构造器给
    • 如果创建了对象，可以在外部给，不给初始值也有缺省值
    • 整型：0，Boolean=false，Human =null;
  • final必须要给初始值-成员变量
    • 定义的时候给初始值
    • 初始化块给初值
    • 构造器给初值
    • 但上面就只能在一个地方定值
  • 静态成员变量
    • 在定义的时候给初始值
    • 在静态初始化块给初值
    • 但上面就只能在一个地方定值
  • 局部变量
    • final修饰的普通局部变量
      • 可以在定义的时候给初值，也可以在定义的时候不给初值，后续赋值语句给值，但一旦给了值就不可以赋给其他值了
    • final修饰的形参
      • 形参的值是通过方法传递的实参值来给的
    • final修饰的引用类型
      • final修饰的引用类型局部变量，一旦指向了某块堆内存，就不允许在指向其他堆内存了，指向位置不能改变，但指向的值可以改变

final宏替换能力

  • 为什么两个字符串String内容相同，比较结果不相同
  • 有的字符串的内容/值在编译的时候就能确定
    • 而result编译的时候确定不了其值，而需要在运行的时候才能确定result3的值
  • Java针对字符串，比如编译期间就能确定字符串内容的，这些内容会被缓存到一个专门的地方去
  • sr = result = result2
  • 如果用final那么：sr = result = result2 = result3 = result4
  • 宏替换在编译就能缓存值

final修饰方法

  • 一旦一个方法被final修饰，那么这个方法就不可以被子类覆盖了
  • private 不要和final 联合修饰，因为private不会被继承

final修饰类

  • 就不能继承这个类了

接口的基本概念

  • 接口：比抽象类还抽象
  • 这里把接口理解为抽象类 —接口理解为类
  • 格式：
    • 【修饰符】 interface 接口名{
    • }
  • 修饰符一般是public，不加就是缺省访问权限

接口的使用

  • 接口也不能实例化

  • 接口内部一般定义一些规范的东西

  • 规定继承者必须要实现的东西

  • 一般是用具体的类去实现这个接口（当父类）。实现这个接口的类，叫做实现类

  • ```java
    public class impint implements kxinterface{ //实现类

    }
    //继承两个类
    public class extends Human impint implements kxinterface{ //实现类

    }
    //继承多个接口
    public class impint implements kxinterface,kxinterface2{ //实现类

    }

  • 接口不能实例化，和抽象类一样

  • 接口可被用于声明[定义] 引用类型变量;多态

接口核心思想和可放入内容

  • 接口一般定义和规范性的东西，但不提供具体的实现。规范和实现分类的设计原则
    • 规范：接口中
    • 实现：在具体的实现类中

接口中不允许出现的内容

  • 构造器
  • 初始化块
  • 接口可以继承多个接口但不可以继承类
  • 接口中不能出现普通成员变量（只有静态成员变量），没有普通方法(有方法体的方法)，只有抽象方法（没用方法体）

共性问题

  • 接口里面大部分都是public权限。所以接口定义的内容可以不写public，但是要写就必须要写成public，写成其他修饰符会报语法错误

接口允许的内容

  • 成员变量（自动变成了静态成员变量）
    • 自动默认加上static 和final
    • static：跟着类走
    • final：必须给这个成员变量指定初始值，而且一旦有初始值就不能修改
  • 普通成员方法（自动变成抽象方法）
    • 不写任何前面的修饰词
    • 自动加上了 abstract，不可以有方法体
    • 必须要实现接口里面的方法
    • 不是静态方法，不能用接口调用，只能用对象（多态（父=子）或（子=子））
  • 静态方法（变成类方法）
    • 跟着类走，用static修饰。
    • 必须要有方法体
    • 直接用接口调用（不能用对象调用）
  • 默认方法（实例方法）
    • 用default修饰，public可以省略，必须要有方法体，否则报错
    • 跟着对象走的，所以不可以和static连用
    • 能写方法体
  • 私有方法
    • 用private修饰
    • 必须要有方法体
    • 跟着对象走（普通私有方法）
    • 跟着类走（加上static）
    • 无法被main调用，作用是仅供接口内部调用
    • 不能被普通方法调用

接口的继承

  • 一个接口多个父类，但接口只能继承接口，不能继承类

接口与抽象类的概括

  • 抽象类是从多个类中抽取出来的模板，接口。接口好像是100%纯抽象类。
  • 接口是一种设计规范，具体的实现是通过继承这个接口的实现类来实现的，私有接口体现的是一种规范和实现分离的逻辑
  • 接口，可以看作一个类，只不过这个类比较特殊
  • 接口的主要途径是作为实现类的父类。并在具体实现类中实现需要的东西，比如接口中的普通方法（抽象方法）

接口和抽象类的一同

  • 两者都不能通过实例化，不如new一个接口，new一个抽象类都不行
  • 抽象类一般通过子类的继承extends来使用
  • 接口通过一个类的实现implements来实现。所以抽象类和接口都是等着别人来几次
  • 有些接口里可以有，抽象类不可以有，反之亦然
  • 接口中出现的很多东西，都是自带的public和static修饰
