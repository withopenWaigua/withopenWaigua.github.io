Java类的继承与super详解

Posted on 2021-12-06 | In Java , 基础内容

如何进行类继承

父亲类、孩子类；

父类【基类、超类】派生出孩子类【派生类、子类】

  • 既有父亲类又有孩子类这种层次关系我们就叫做继承，孩子类能从父亲类哪里继承到许多东西
  • 我们可以把一些通用的东西放在父类，然后每个孩子可以派生出自己独特的东西，还可以从父亲哪里继承出一些通用的东西，防止子类中出现很多重复的代码

继承

  1. 先定义一个父类
  2. 父类定义一些公共的成员变量、方法
  3. 通过继承这个父类来构建新的子类
  • 通常来说，子类会比他的父类有更多的成员变量以及方法，子类一般比父类更庞大【内容更多，占用更多内存】

子类继承父类的语法格式：

修饰符 class 子类名 extends 父类名{

//成员变量

//成员方法

}

  • extends 扩展
  • 只能继承一个父类
  • 继承后，子对象包含了父类的对象

覆盖父类的方法

  • 子类重写一个参数相同的方法（必须返回值相同）
  • 如果参数不同【数量不同，类型不同】， 那么就不会覆盖
  • 静态方法也一样用静态方法去覆盖

调用父类同名函数的方法

  • super - 》理解成指向当前对象的父亲子对象的箭头或指针，super一般也只出现在类的方法中
  • this -》指向堆中的自己
  • 静态方法中不能出现this，super在静态方法中也不能够使用
  • 有对象才有this、super

覆盖父类的成员变量

  • 在java的类继承中，子类能覆盖父类的同名变量

如果覆盖后还要使用父类的成员变量

  • 强制转换
  • 可以强转调用父类中的成员变量 /静态成员变量/被覆盖的静态方法/未被覆盖后的普通方法
  • 但不能强转成覆盖后的父类的普通方法

构造器

  • 定义并初始化一个对象时候

  • 父类构造器先执行

  • 再执行了子类构造器

  • public class Human {
    //构造器
    Human(int i){
        System.out.println("执行了Human::human()构造器");
    }
public class Men extends Human{
    Men(){
        super(12); //因为父类构造器有了参数，无法使用默认构造器，因此要子类传参数进去调用父类构造器，还必须写道第一行
        System.out.println("执行了Men()构造器");
    }

Men mymen = new Men();

结论：

  1. 不管是否第一行使用了super（）来调用父类构造器，在初始化一个Men对象的时候，父类和子类构造器都只执行一次
  2. 不写的时候默认调用父类的默认构造器，要调用其他构造器就必须写super（XXXX）
  3. 既然可以通过super（）调用父类构造器，可以推出this可以调用本类的其他构造器
  4. super和this不能同时出现，因为都必须放在子类构造器第一行
  5. 如果子类构造器没用super和this，先调用父类构造器，再调用子类构造器
